### v1.5.41 (2025-01-13) - ⚡ loadSettings 최적화: 캐시 UI 업데이트 및 필드별 업데이트
**상태**: ✅ 개발 완료

**주요 개선사항**:
- 🔄 캐시에서 가져온 경우에도 UI 업데이트: 캐시된 데이터를 사용할 때도 UI가 정상적으로 표시되도록 수정
- ⚡ 필드별 선택적 업데이트: `fields` 파라미터에 따라 필요한 필드만 DOM 업데이트 (불필요한 작업 제거)
- 🐛 기본 정보 재진입 시 안 보이는 문제 해결: 캐시에서 가져온 경우에도 `renderBasicSummary` 호출

**해결된 문제**:
- ✅ 기본 정보가 한 번 불러와서 제대로 보였는데 다시 메뉴에 들어가면 내용이 안 나오는 문제 해결
- ✅ 정보가 대단한 게 없는데 로딩이 오래 걸리는 문제 해결 (필드별 선택적 업데이트)
- ✅ 불필요한 DOM 조작 제거로 성능 개선

**기술적 변경**:
- `loadSettings` 함수 개선:
  - 캐시에서 가져온 경우에도 UI 업데이트 로직 실행
  - `fields` 파라미터에 따라 필요한 필드만 업데이트 (`shouldUpdateField` 함수 추가)
  - `basic`, `delivery`, `discount`, `pickup`, `businessHours`, `images`, `sectionOrder` 필드별로 조건부 업데이트
- 불필요한 DOM 조작 제거:
  - 요청하지 않은 필드에 대한 DOM 업데이트 제거
  - 예: `loadSettings(['basic', 'businessHours'])` 호출 시 배달앱 설정, 할인 설정 등은 업데이트하지 않음

**성능 개선 효과**:
- 불필요한 DOM 조작 제거로 로딩 시간 단축
- 필드별 선택적 업데이트로 CPU 사용량 감소
- 캐시 사용 시에도 UI가 정상적으로 표시되어 사용자 경험 개선

### v1.5.40 (2025-01-13) - 📊 로딩바 개선: 한 번에 표시 및 연속 진행률
**상태**: ✅ 개발 완료

**주요 개선사항**:
- 🔄 로딩바 한 번 표시: 전체 화면 로딩바가 한 번만 표시되고 모든 작업이 완료될 때까지 유지
- 📈 연속 진행률: 진행률이 0%에서 100%까지 연속적으로 증가 (깜빡임 없음)
- 📝 로딩 텍스트 업데이트: 현재 진행 중인 작업을 텍스트로 표시
- 🚫 API 요청 레벨 로딩바 제거: `apiRequest`에서 로딩바 표시/숨김 제거 (섹션 레벨에서만 관리)

**해결된 문제**:
- ✅ 로딩바가 올라가다가 다시 줄어들었다가 다시 로딩되는 문제 해결
- ✅ 전체 화면 로딩바가 한 번에 보이도록 개선
- ✅ 진행률이 연속적으로 증가하도록 개선 (0% → 100%)
- ✅ 뭐가 로딩되는지 텍스트로 명확히 표시

**기술적 변경**:
- 로딩바 관리자 개선:
  - `isLoadingActive` 플래그로 로딩바 활성화 상태 추적
  - 첫 번째 요청일 때만 모달 표시 및 진행률 초기화
  - 이미 로딩 중이면 텍스트만 업데이트
  - 진행률 애니메이션을 연속적으로 실행 (깜빡임 없음)
- `apiRequest` 함수 수정:
  - 로딩바 표시/숨김 로직 제거
  - 섹션 레벨에서만 로딩바 관리
- 섹션별 데이터 로드:
  - 각 섹션에서 로딩바를 한 번만 표시
  - 모든 작업이 완료된 후 한 번만 숨김
  - 에러 발생 시에도 적절히 처리

**사용자 경험 개선**:
- 로딩바가 깜빡이지 않고 연속적으로 진행
- 현재 진행 중인 작업을 텍스트로 명확히 표시
- 전체 화면 로딩바가 한 번에 표시되어 일관된 경험 제공

### v1.5.38 (2025-01-13) - ⚡ 세션 캐싱 및 로딩 진행률 표시
**상태**: ✅ 개발 완료

**주요 개선사항**:
- 💾 세션 스토리지 캐싱: 5분 TTL로 데이터 캐싱 (불필요한 API 호출 방지)
- 📊 로딩 진행률 표시: 로딩 모달에 진행률 바와 퍼센트 표시
- ⚡ 초기 로드 최소화: 새로고침 시 홈 페이지만 표시하고 데이터는 필요할 때만 로드
- ⏱️ 타임아웃 단축: API 요청 타임아웃을 10초에서 8초로 단축

**해결된 문제**:
- ✅ 새로고침 후 모든 데이터를 불러오는 문제 해결 (홈 페이지만 표시)
- ✅ 로딩 진행률이 없어 얼마나 기다려야 하는지 모르는 문제 해결
- ✅ 1분 이상 로딩바만 보이는 문제 해결 (타임아웃 및 캐싱)
- ✅ 세션을 적극적으로 활용한 최적화 구현

**기술적 변경**:
- 세션 스토리지 캐싱 유틸리티 추가:
  - `window.sessionCache.set(key, data)`: 데이터 캐싱 (5분 TTL)

  - `window.sessionCache.clear(key)`: 캐시 삭제
- `loadSettings()` 함수에 캐싱 로직 추가:
  - 캐시 키: `settings_{storeId}_{fields}`
  - 캐시에서 데이터가 있으면 즉시 반환
  - API 호출 후 캐시에 저장
- 로딩 모달에 진행률 표시 추가:
  - 진행률 바 (0-100%)
  - 퍼센트 텍스트 표시
  - 진행률 시뮬레이션 (최대 90%까지)
- `DOMContentLoaded` 이벤트 최적화:
  - 홈 페이지만 표시
  - 가게 목록만 로드
  - 슈퍼어드민 정보, 활동 로그, 점주 계정 등은 필요할 때만 로드
  - 사용자 정보 로드는 제거 (섹션 클릭 시에만 로드)
- API 타임아웃 단축: 10초 → 8초

**성능 개선 효과**:
- 초기 로딩 시간 대폭 단축 (홈 페이지만 표시)
- 세션 캐싱으로 반복 요청 방지
- 로딩 진행률 표시로 사용자 경험 개선

## 🎯 버전 명명 규칙

### Semantic Versioning (SemVer)
**형식**: `MAJOR.MINOR.PATCH`

- **MAJOR**: 호환성이 깨지는 큰 변경
- **MINOR**: 하위 호환성을 유지하는 기능 추가
- **PATCH**: 하위 호환성을 유지하는 버그 수정

### 예시
- `1.0.0` → `1.1.0`: 새로운 기능 추가 (하위 호환)
- `1.1.0` → `1.1.1`: 버그 수정 (하위 호환)
- `1.1.1` → `2.0.0`: 큰 변경 (호환성 깨짐)
